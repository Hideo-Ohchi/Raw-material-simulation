# デバッグ版VBA実行手順

## ⚙️ デバッグ情報を有効にしたVBAを埋め込みました

修正内容：
- 設定シートから読み込んだ値を詳細に表示
- 月列を完了メッセージに表示
- 生産計画から抽出したデータ件数を表示

---

## 🔍 実行手順

### ステップ1: 設定シートの値を確認・変更

```
1. Excel 原料シミュレーション.xlsm を開く
2. 「設定」シートを表示
3. 以下の値を設定：
   ├─ A1: 「開始年月」 → B1: 202603
   ├─ A2: 「計画月数」 → B2: 4
```

**現在の状態を確認してください**

### ステップ2: VBAコンソールを開く（オプション）

VBAの内部ログを確認したい場合：

```
1. Excel で Alt+F11
2. VBA編集画面を開く
3. Ctrl+G でコマンドウィンドウを開く
4. この状態でマクロを実行すると、Debug.Print の出力が表示される
```

### ステップ3: マクロを実行

```
1. Alt+F8 キー
2. マクロ一覧から「BuildSimulationInput」を選択
3. [実行] をクリック
4. 完了メッセージを確認
```

## 🐛 デバッグ時の確認ポイント

完了メッセージで以下を確認：

| 項目 | チェック点 |
|------|----------|
| **設定値** | B1、B2が正しく読み込まれているか |
| **月列** | 設定値に基づいた正しい月が表示されているか |
| **月数** | 計画月数の分だけ月が表示されているか |
| **データ** | 期待される品目数が抽出されているか |
| **行数** | ヘッダー1 + 品目数 = 全行数か |

## 🔧 問題時の確認項目

### Q: 「完了メッセージの月列が期待と違う」

**確認**:
1. 設定シートで B1, B2 を確認
2. セルがテキスト形式になっていないか確認
   - 数値形式に設定する必要があります
3. セルに空白スペースが含まれていないか確認

### Q: 「生産計画から抽出されたデータが0」

**確認**:
1. 生産計画シートが存在するか確認
2. 生産計画の列構成を確認
   - A列: 品目コード
   - B列: 対象年月（YYYYMM形式）
   - C列: 計画数量
3. データが対象期間内か確認

## 💡 デバッグコンソールの使い方

VBA編集画面のコマンドウィンドウで：

```
1. Alt+F11 を押す
2. メニュー: 表示 → イミディエイトウィンドウ
3. 下部にコンソール画面が表示される
4. BuildSimulationInput を実行するとDebug.Printの出力が表示される
```

出力例：
```
=== BuildSimulationInput デバッグ情報 ===
B1（設定シート）の値: 202603 -> startYM: 202603
B2（設定シート）の値: 4 -> monthCount: 4
月列: 202603, 202604, 202605, 202606
生産計画から抽出したデータ件数: 79 品目
シミュレーション入に書き込んだ行数: 79 品目
```
