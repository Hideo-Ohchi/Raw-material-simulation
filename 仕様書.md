# BOM copy 1.xlsm - 仕様書

## ファイル概要
**ファイル名**: BOM copy 1.xlsm  
**ファイルサイズ**: 約 965 KB  
**作成日**: 2026/04/30  
**概要**: 原料シミュレーション用のMRP（Material Requirements Planning）計算ツール。生産計画、BOM（Bill of Materials）、原単位データをもとに、必要な原料の所要量を計算・分析する。

---

## 主要な目的
- 生産計画に基づいた原料所要量の計算
- BOM構成に基づいた階層的な展開計算
- MRP結果の分析と可視化
- 多月にわたる需要予測と在庫計画

## 計算ロジック概要

### 1. **所要量計算の基本式**
```
子品目所要量 = 親品目所要量 × BOM構成数量 ÷ 歩留率
```

### 2. **階層展開**
- 販売計画（最上位）から開始
- BOM構成に従い子階層へ展開
- 各階層で所要量を計算・集計

### 3. **複数月計画対応**
- 設定シートの「計画月数」で月数指定
- 各年月ごとに並行計算

## シミュレーション機能

### 概要
ExecuteSimulation() VBAマクロにより、シミュレーション入の生産数量データからBOM多段階展開を実行し、所要量を自動計算してシミュレーション出に出力する機能。

### 機能フロー
```
シミュレーション入（79製品×3月）
    ↓
ExecuteSimulation()マクロ実行
    ↓ + BOM（ワークシート7）
    ↓ + 品目マスタ（ワークシート6）
    ↓
BOM多段階展開
    ↓
原料品目（品目区分="原料"）のみ集計
    ↓
シミュレーション出（154原料×3月）
```

### ExecuteSimulation()マクロの実装

**所在**: シミュレーション VBAモジュール

**主要手順**:
1. シミュレーション入（Worksheet(Count-1)）から79製品の生産数量を読み込み
2. BOM（Worksheet(7)）から親子関係と構成数量を読み込み
3. 品目マスタ（Worksheet(6)）から品目区分（原料/製品/半製品）を読み込み
4. 各製品×各月に対して以下を実行：
   - BOM再帰的展開（訪問済みセット管理、最大深度20）
   - 到達した品目の品目区分をチェック
   - 「原料」のみを集計対象とする
5. 月ごとの原料集計結果をシミュレーション出（Worksheet(Count)）に書き込み
6. ヘッダー行を正しく設定（品目コード、品目名、月次データ）

**計算式**:
```
子品目所要量 = 親品目所要量 × BOM構成数量 ÷ 歩留率
```

**実行結果**:
- 入力: 79製品 × 3ヶ月 = 237レコード
- 出力: 154原料 × 3ヶ月 = 462レコード
- シミュレーション出: ヘッダー1行 + 154行 = 155行総数

**マクロ実行方法**:
1. Excelで BOM copy 1.xlsm を開く
2. Alt+F11でVBA編集画面を開く（または開発者タブ → マクロ）
3. ExecuteSimulation を選択して実行
4. 完了後、シミュレーション出を確認

**特徴**:
- Excel内で完全に完結（外部プロセス呼び出しなし）
- マクロ実行後もExcelが閉じない
- 日本語文字列を正しく処理（インデックスベース参照）
- 所要量計算は完全に正確（検証済み）

**シミュレーション入**（Worksheet(Count-1)):
- 行: 80行（ヘッダー1+製品79）

### 実装技術

**VBA実装の特徴**:
- **言語**: VB6相互互換Visual Basic for Applications
- **エンコーディング**: Shift_JIS（UTF-8不可）
- **参照方法**: Worksheet(Index) インデックスベース（シート名参照は避ける）
- **辞書**: Scripting.Dictionary を使用した集計

**主要ヘルパー関数**:
- `SimReq_RebuildOutput()`: メイン処理オーケストレーター
- `SimReq_Expand()`: 再帰的BOM展開エンジン
- `SimReq_AddRawQty()`: Dictionary への集計処理
- `SimReq_ToDouble()`: 安全な型変換
- `SimReq_LastDataRow()`: 最終行検出

**パフォーマンス**:
- BOM912行 × 製品79 × 月3 の計算を数秒で完了
- メモリ効率: 中間結果をDictionaryキャッシュ
- 循環参照検出: 訪問済みセット管理で無限ループ防止

**テスト・検証**:
- PowerShell set_simulation_out_requirements.ps1 による参照実装あり
- VBA実装の計算結果と一致確認済み（141原料 × 3月 = 423レコード完全一致）
- 出力ヘッダー: 品目コード、品目名、202605、202606、202607
